# STM8L001 + CR2032 3键 NEC 红外遥控器完整设计方案

## 1. 文档目的

基于以下已确定前提，输出一份可直接用于原理图、PCB 和固件实现的完整设计方案：

- MCU：`STM8L001J3M3`
- 供电：`CR2032`
- 红外协议：标准 `NEC`
- 固定码值：

| 按键 | 用户码 | 用户反码 | 按键码 | 按键反码 | 完整帧 |
|---|---:|---:|---:|---:|---|
| KEY1 | `0x80` | `0x7F` | `0xC0` | `0x3F` | `80 7F C0 3F` |
| KEY2 | `0x80` | `0x7F` | `0x40` | `0xBF` | `80 7F 40 BF` |
| KEY3 | `0x80` | `0x7F` | `0x60` | `0x9F` | `80 7F 60 9F` |

本方案目标不是生成完整源码或 EDA 库，而是提供一套**工程上稳妥、调试友好、可直接开工**的硬件与软件设计基线。

## 2. 设计目标与边界

## 2.1 设计目标

1. 用 `STM8L001J3M3` 直接产生指定 NEC 红外波形。
2. 使用 `CR2032` 供电，待机功耗尽量低，发射时保持保守但足够的红外余量。
3. 第一版优先“**调试与实现稳妥**”，不是极限压低 BOM 成本或极限压缩尺寸。
4. 方案交付后，硬件工程师可直接画图，固件工程师可直接开始写驱动与状态机。

## 3. 官方器件事实与工程结论

## 3.1 STM8L001J3M3 的本任务相关事实

结合 ST 官方资料，本任务直接相关的事实如下：

1. `STM8L001J3` 工作电压范围为 `1.8 V ~ 3.6 V`，适合 `CR2032` 直供电。
2. 片内有 `16 MHz` RC 振荡器，且有低功耗模式，适合无外部晶振的遥控器方案。
3. 芯片带有 `IRTIM` 红外接口，`TIM2` 用于载波，`TIM3` 用于包络调制。
4. `PA0/SWIM/BEEP/IR_TIM` 复用在同一个引脚上，因此 **SWIM 与 IR 输出必须在系统设计中一起考虑**。
5. 本器件**没有专用 NRST 引脚**，通过 SWIM 烧录/调试时只能采用 `connect-on-the-fly` 思路，不能依赖 `connect-under-reset`。
6. 数据手册明确建议：如果 `PA0` 在运行时要从 SWIM 切到普通 I/O/IR 功能，固件上电后应保留约 `5 s` 的 SWIM 接入窗口，否则器件可能难以再次接入调试。
7. 参考手册指出可以打开 `HS_EN` 让 `PA0` 直接高灌电流驱动 IR LED，但该模式下其它 I/O 电平不能保证，因此**不适合作为首版调试稳妥方案默认路径**。

## 3.2 工程结论

基于上面约束，本项目第一版采用以下原则：

- **IR LED 不采用 PA0 直驱**，改用 `PA0 -> 外部 NMOS -> IR LED` 的保守拓扑；
- **PA0 同时兼容 SWIM**，通过“调试固件保留 SWIM 时间窗 + 板上测试点”保证可重新烧录；
- 键盘输入使用独立 GPIO + 内部上拉；
- 不使用外部晶振；
- 功耗优化以“足够低 + 易调试 + 不锁死器件”为优先。

## 4. 系统总体方案

## 4.1 系统框图

```text
CR2032
  |
  +-- VDD/VSS 去耦网络
  |
  +-- STM8L001J3M3
  |     |
  |     +-- KEY1 输入
  |     +-- KEY2 输入
  |     +-- KEY3 输入
  |     +-- PA0/SWIM/IR_TIM -> 栅极电阻 -> NMOS -> IR LED
  |     +-- 可选 DBG_STRAP / 预留脚
  |
  +-- 调试/测试点：VDD, GND, SWIM(IR), IR_LED_K, 可选 DBG_STRAP
```

## 4.2 设计策略

- `TIM2`：生成 `38 kHz` 载波；
- `TIM3`：生成 NEC 包络（560 us / 1.69 ms / leader / repeat）；
- `IRTIM`：把 TIM2 载波与 TIM3 包络合成为 `PA0` 上的红外输出；
- `TIM4`：1 ms 级去抖/软件定时（可选，但推荐保留）；
- `EXTI`：按键唤醒；
- `Halt`：空闲时最低功耗待机。

## 5. 推荐引脚分配

STM8L001J3M3 为 SO-8，小封装但有 6 个可用 I/O 焊盘。结合调试优先级，推荐引脚分配如下。

## 5.1 推荐分配表（第一版默认）

| 物理脚 | 复用脚名 | 第一版用途 | 说明 |
|---|---|---|---|
| 1 | `PA0 / SWIM / IR_TIM / PC3 / PC4` | `IR 输出 + SWIM` | 最关键引脚；调试阶段上电后保留 SWIM 窗口，再切 IR 输出 |
| 2 | `PA2 / PA4 / PA6` | `KEY1` | 普通数字输入，内部上拉，按键到 GND |
| 3 | `VSS` | GND | 电池负极 |
| 4 | `VDD` | 3 V 电源 | CR2032 正极 |
| 5 | `PD0 / TIM3_CH2 / PB3 / TIM2_ETR` | `KEY2` | 普通数字输入，内部上拉，按键到 GND |
| 6 | `PB6 / SPI_MOSI` | `KEY3` | 普通数字输入，内部上拉，按键到 GND |
| 7 | `PB7 / SPI_MISO / PC0` | `预留/调试辅助` | 首版建议预留测试焊盘，可做调试模式输入或产测辅助 |
| 8 | `PC1 / I2C_SCL / PC2 / USART_RX` | `DBG_STRAP 或预留` | 建议预留到测试焊盘，作为“强制保留 SWIM”或版本分支输入 |

## 5.2 为什么不把全部脚都用满

因为用户优先级是“调试与实现稳妥优先”，所以第一版刻意保留 1~2 个弹性脚位：

- 方便加入 `DEBUG strap`；
- 方便做工厂测试或异常恢复；
- 避免 SO-8 小资源器件一开始就把全部脚占死，后续难调。

如果后续确认无需 `DBG_STRAP`，则 pin7/pin8 可在量产版中重新利用。

## 6. 硬件设计方案

## 6.1 电源方案

### 6.1.1 电源拓扑

- `CR2032` 电池直接供电到 `VDD`
- MCU 不加 LDO
- 靠近 MCU 放置：
  - `C1 = 100 nF` 去耦
  - `C2 = 4.7 µF ~ 10 µF` 主去耦
- 靠近红外发射支路预留：
  - `C3 = 22 µF ~ 47 µF` 可选储能电容（建议首版预留焊盘）

### 6.1.2 电源设计理由

- `CR2032` 的平均供电能力足够，但对脉冲大电流较敏感；
- 红外发射是短脉冲行为，不建议直接按“连续大电流负载”设计；
- 预留 `22~47 µF` 本地储能焊盘，可在样机阶段根据发射距离和波形掉压情况选择是否装配；
- 不推荐串联二极管做反接保护：会降低电压余量，直接影响红外峰值电流与低电量工作边界；
- 电池座本身已具备较强防呆特性，第一版可不加额外反接器件。

## 6.2 按键输入

### 6.2.1 连接方式

每个按键采用：

```text
GPIO ---- 按键 ---- GND
```

MCU 内部开启上拉。

### 6.2.2 推荐设计

- 不加外部上拉，减少 BOM；
- 不加硬件 RC 去抖，先由软件去抖；
- 每个按键走线尽量短，靠近 MCU；
- 若 ESD 环境偏恶劣，可在后续版本中增加串联小电阻（`33~100 Ω`）或 TVS，但第一版不默认加。

## 6.3 红外发射级（默认推荐方案）

## 6.3.1 拓扑

推荐使用 **外部 NMOS 低边开关**：

```text
VBAT(3V)
  |
  +-- R_LED -- IR_LED --+-- D(Q1 NMOS)
                        |
                     S(Q1) -> GND

PA0(IR_TIM/SWIM) -- Rg(100R) -- G(Q1)
                      |
                    Rpd(100k)
                      |
                     GND
```

### 推荐器件

- `Q1`：小信号逻辑级 NMOS，推荐 `AO3400A` / `SI2302` / 同级替代
- `R_LED`：首版建议 `33 Ω ~ 47 Ω`，默认先装 `39 Ω`
- `Rg`：`100 Ω`
- `Rpd`：`100 kΩ`
- `IR_LED`：940 nm 红外 LED，推荐 5 mm 高辐射型，或贴片高功率型一颗

## 6.3.2 为什么不用 MCU 直驱

虽然 STM8L001 的 `PA0` 支持 `HS_EN` 直接下拉较大电流，但参考手册已经给出警告：

- 高灌电流模式开启时，其它 I/O 电平不能保证；
- 这和“调试与实现稳妥优先”的目标冲突；
- 一旦 PA0 上同时承载 SWIM 和直驱 LED，大电流切换对调试窗口、串扰和恢复都不友好。

因此：

- **首版默认禁用 `HS_EN`**；
- `IR_CR` 只打开 `IR_EN`，不打开 `HS_EN`；
- 大电流由外部 NMOS 承担。

## 6.3.3 峰值电流建议

以 `VBAT = 3.0 V`、单颗 IR LED 正向压降约 `1.2~1.4 V`、NMOS 压降较小估算：

- `R_LED = 39 Ω` 时，峰值电流约在 `35~45 mA` 量级；
- 对单颗 CR2032 来说，这是一个**保守但可用**的首版目标；
- 如果样机实测距离不足，可优先尝试：
  1. 装配 `C3` 储能；
  2. 把 `R_LED` 下调到 `33 Ω`；
  3. 换更高辐射 IR LED；
- 不建议第一版就把峰值设计到 `80~100 mA`，否则容易把 CR2032 脉冲能力假设得过于激进。

## 6.4 SWIM 调试接口

### 6.4.1 关键事实

STM8L001J3M3 **没有独立 NRST 引脚**。因此调试建议必须修正为：

- 最低需要测试点：`VDD`、`GND`、`SWIM(PA0)`
- 不依赖 `connect-under-reset`
- 使用 `connect-on-the-fly`

### 6.4.2 推荐测试点

| 测试点 | 是否必留 | 用途 |
|---|---|---|
| `TP_VDD` | 必留 | 给 ST-LINK 参考电压 |
| `TP_GND` | 必留 | 地参考 |
| `TP_SWIM` | 必留 | SWIM 编程与调试 |
| `TP_IR` | 建议 | 示波器观察 IR 调制输出（可放在 NMOS 栅极或漏极） |
| `TP_DBG` | 建议 | pin7/pin8 预留测试点，用于调试模式/工厂模式 |

### 6.4.3 调试固件建议

调试版固件上电后建议：

1. 初始化最基本时钟；
2. 检查 `DBG_STRAP` 是否被拉到特定状态；
3. 若处于调试版，保留约 `5 s` SWIM 窗口，不立即将 `PA0` 切到 IR 功能；
4. 若检测到 `DBG_STRAP` 请求，可一直保持 SWIM 功能并禁止发射；
5. 正常模式下，时间窗结束后再开启 `IR_EN`。

量产锁定版可去掉 5 s 延时，但建议至少保留 `DBG_STRAP` 恢复路径。

## 6.5 未用脚处理

STM8L001 有多个内部未引出 GPIO，参考资料建议把未使用 GPIO 配置为**推挽输出低电平**以降低功耗并改善 EMC。

对外引出的但当前未使用的 pin7/pin8：

- 若完全不用，建议配置为**输入上拉**或**推挽低**，二选一并保持一致；
- 若要作调试 strap，配置为**输入上拉**，外部调试时短接到 GND 即可。

## 8. 功耗与电池寿命预算

## 8.1 待机功耗目标

参考 STM8L001J3 产品资料，芯片在 Halt 模式下可到亚微安级。考虑实际板级泄漏和预留脚处理，第一版建议工程目标为：

- MCU + 板级待机电流目标：`< 2 µA`
- 较差但可接受目标：`< 5 µA`

影响待机功耗的主要因素：

1. 未用脚是否浮空；
2. 外部储能电容和 MOSFET 漏电；
3. 键盘上拉方式；
4. 是否留有外挂调试器/测试夹具。

## 8.2 单帧 NEC 发射时间估算

标准 NEC 帧总时长近似为：

- Leader：`9.0 ms + 4.5 ms`
- 32 bit 数据：
  - 每 bit 固定 `560 us` 载波
  - 逻辑 0：后跟 `560 us` 空闲
  - 逻辑 1：后跟 `1.69 ms` 空闲
- Stop bit：`560 us`

对于“8 位原码 + 8 位反码”的地址和数据结构，一个字节与其反码总是 8 个 1、8 个 0，因此本方案三个键的完整帧总时长可近似按相同值估算：

- 32 bit 内总载波时间：`32 × 0.56 ms = 17.92 ms`
- 数据区空闲时间：`16 × 0.56 ms + 16 × 1.69 ms = 36.00 ms`
- 总帧长：`9 + 4.5 + 17.92 + 36 + 0.56 ≈ 67.98 ms`

即：**一次完整发送约 68 ms**。

## 8.3 发射平均电流估算

假设：

- IR 峰值电流：`40 mA`
- 载波占空比：约 `1/3`
- 整帧载波总占比：`27.48 ms / 67.98 ms ≈ 40.4%`
- MCU 发射期平均电流：`~2 mA`

则 IR LED 在整帧上的平均电流约：

```text
I_IR_avg_frame ≈ 40 mA × 1/3 × 27.48 / 67.98 ≈ 5.4 mA
```

整机在一帧期间的平均电流可粗估为：

```text
I_total_frame ≈ 5.4 mA + 2 mA ≈ 7.4 mA
```

一次完整帧消耗电量约：

```text
Q_frame ≈ 7.4 mA × 0.068 s = 0.5032 mA·s
       ≈ 0.00014 mAh
```

## 8.4 使用寿命估算（工程级）

取保守可用容量 `150 mAh`（不是理想 220 mAh），估算如下：

### 场景 A：轻度使用（20 次/天）

- 发射消耗：`20 × 0.00014 ≈ 0.0028 mAh/天`
- 待机消耗（2 µA）：`0.048 mAh/天`
- 合计：`~0.051 mAh/天`
- 理论寿命：`150 / 0.051 ≈ 2941 天 ≈ 8.0 年`

### 场景 B：中度使用（100 次/天）

- 发射消耗：`0.014 mAh/天`
- 待机消耗：`0.048 mAh/天`
- 合计：`~0.062 mAh/天`
- 理论寿命：`150 / 0.062 ≈ 2419 天 ≈ 6.6 年`

### 场景 C：重度使用（300 次/天）

- 发射消耗：`0.042 mAh/天`
- 待机消耗：`0.048 mAh/天`
- 合计：`~0.090 mAh/天`
- 理论寿命：`150 / 0.090 ≈ 1667 天 ≈ 4.6 年`

## 8.5 对寿命估算的正确理解

上面数据的意义是：

- 对 CR2032 遥控器，**待机功耗往往比按键使用频率更主导寿命**；
- 真正限制体验的，往往不是“容量先耗尽”，而是：
  - 电池老化后内阻上升；
  - 发射瞬间压降变大；
  - 红外距离先下降；
- 所以本项目首版的关键不是追求理论 8 年寿命，而是保证：
  1. 待机足够低；
  2. 低电量时仍能稳定发射；
  3. 调试阶段可以方便观察压降与峰值电流。

## 9. 固件总体架构

## 9.1 模块划分

建议固件拆分为以下模块：

1. `bsp_clock`：时钟初始化
2. `bsp_gpio`：按键/测试脚/SWIM 相关 GPIO 初始化
3. `bsp_ir`：TIM2 + TIM3 + IRTIM 初始化与发射控制
4. `key_scan`：按键扫描与去抖
5. `remote_proto`：NEC 编码打包
6. `power_mgr`：睡眠/唤醒控制
7. `app_main`：主状态机

## 9.2 系统状态机

推荐状态机如下：

```text
上电复位
  -> 调试窗口/恢复模式判断
  -> 外设初始化
  -> 待机(Halt)
      -> 按键外部中断唤醒
      -> 去抖确认
      -> 判定按键
      -> 发送完整 NEC 帧
      -> 若长按成立，周期性发送 Repeat 帧
      -> 按键释放
      -> 回到 Halt
```

## 9.3 上电策略

### 调试版固件

- 上电先不开启 IR 输出
- 保留 SWIM 时间窗约 `5 s`
- 期间可通过 `DBG_STRAP` 进入维护模式
- 超时后再切换 `PA0` 到 IRTIM 输出

### 量产版固件

- 可移除 5 s 固定等待
- 但建议保留一种恢复入口，例如：
  - `DBG_STRAP` 拉低时进入维护模式
  - 或上电后首次按住某组合键进入维护模式

## 10. NEC 编码与发送策略

## 10.1 基本时序

采用标准 NEC：

- Leader mark：`9.0 ms`
- Leader space：`4.5 ms`
- Bit mark：`560 us`
- 逻辑 0：`560 us mark + 560 us space`
- 逻辑 1：`560 us mark + 1690 us space`
- Stop bit：`560 us mark`
- Repeat frame：`9.0 ms mark + 2.25 ms space + 560 us mark`
- Repeat 周期：约 `108 ms`

## 10.2 发送顺序

每个字节按 `LSB first` 发送。

建议把每个键的发送帧在 ROM 中直接表述为：

```text
KEY1 -> addr=0x80, addr_inv=0x7F, data=0xC0, data_inv=0x3F
KEY2 -> addr=0x80, addr_inv=0x7F, data=0x40, data_inv=0xBF
KEY3 -> addr=0x80, addr_inv=0x7F, data=0x60, data_inv=0x9F
```

而不是运行时每次重新查表复杂拼装，首版保持逻辑直白可查。

## 10.3 长按策略

推荐行为：

- 按键稳定按下后先发**完整帧** 1 次；
- 若按键继续保持，则每约 `108 ms` 发 1 次 NEC Repeat 帧；
- 若接收端不识别 Repeat，可在第二版切换为周期性重发完整帧，但第一版优先遵守标准 NEC Repeat。

## 10.4 多键同时按下策略

为了避免歧义，第一版建议采用**单键优先策略**：

- 若同时检测到多个按键按下，则判为无效；
- 必须全部释放后重新识别；
- 若业务必须允许多键，则以后续版本再定义优先级。

这样最稳妥，也最利于调试。

## 11. 定时器与寄存器设计建议

## 11.1 推荐资源分工

| 资源 | 用途 | 说明 |
|---|---|---|
| `TIM2 OC1` | 38 kHz 载波 | IRTIM 的高频载波源 |
| `TIM3 OC1` | NEC 包络 | 控制 mark / space 输出门控 |
| `IRTIM (IR_CR)` | IR 输出使能 | 输出到 `PA0` |
| `TIM4` | 去抖/软件节拍（推荐） | 1 ms Tick 或短周期轮询 |
| `EXTI` | 按键唤醒 | 低功耗唤醒入口 |

## 11.2 38 kHz 载波配置示例

若系统时钟使用内部 `16 MHz HSI`，可采用：

```text
TIM2 时钟 = 16 MHz
PSC = 0
ARR = 420
频率 = 16,000,000 / (420 + 1) ≈ 38.0 kHz
```

占空比约 1/3，可设：

```text
CCR1 ≈ 140
占空比 ≈ 140 / 421 ≈ 33.3%
```

这组值简单、直观、便于示波器核对。

## 11.3 IRTIM 控制建议

`IR_CR` 关键位：

- `IR_EN = 1`：开启红外输出
- `HS_EN = 0`：第一版默认关闭高灌电流直驱模式

因此首版建议：

```text
IR_CR = 0b00000001
```

## 11.4 TIM3 包络控制建议

TIM3 负责输出 NEC 的包络门控：

- Leader mark 期间：允许 TIM2 载波通过
- Space 期间：关闭输出
- 逻辑 0 / 1 的空间宽度由 TIM3 重装计数值决定

实现方式建议：

1. TIM2 连续运行 38 kHz PWM
2. TIM3 每次装载下一个 mark/space 时长
3. 在 TIM3 中断里推进 NEC 状态机
4. 发送完成后关闭 `IR_EN` 或关闭包络输出

这样载波和协议时间分离，逻辑更清楚，调试也更容易。

## 11.5 TIM4 去抖建议

- Tick 周期：`1 ms`
- 去抖判定：连续 `10~20 ms` 稳定才确认按下
- 长按 repeat 启动门限：`> 300 ms`
- Repeat 周期：按 NEC 标准约 `108 ms`

TIM4 也可以不用中断，而改为唤醒后短轮询。但首版建议保留 TIM4，逻辑更清晰。

## 12. 软件流程建议

## 12.1 初始化流程

```text
main()
  -> 关中断
  -> 时钟初始化（HSI）
  -> GPIO 默认安全态
  -> 读取 DBG_STRAP
  -> 若为调试版，保留 SWIM 接入窗口
  -> 初始化 TIM2 / TIM3 / TIM4 / EXTI
  -> 开中断
  -> 进入待机
```

## 12.2 待机/唤醒流程

```text
待机前：
  - 关闭 IR_EN
  - 关闭不必要外设时钟
  - 清中断标志
  - 进入 Halt

唤醒后：
  - 确认唤醒源为按键
  - 执行去抖
  - 若确认有效，进入发送流程
  - 发送结束后回待机
```

## 12.3 发送流程

```text
send_key(key_id)
  -> 装载 key 对应 4 字节帧
  -> 发送 leader
  -> 依次发送 32 bit
  -> 发送 stop bit
  -> 若键仍按下，转 repeat 发送
  -> 释放后退出
```

## 12.4 异常保护建议

- 若在发射过程中检测到电压明显跌落（后续版本可加 ADC 的器件才做），则提前停止 repeat；
- 第一版 STM8L001 无 ADC，不做实时电池采样；
- 因此第一版更需要通过保守峰值电流 + 充足去耦来保证边界稳定。

## 13. 原理图级连接建议

## 13.1 最小可用连接

```text
BT1 (+) -------------------------- VDD (pin4)
BT1 (-) -------------------------- VSS (pin3)

VDD -- C1 100nF -- GND
VDD -- C2 4.7uF  -- GND
VDD -- C3 22~47uF(optional) -- GND

pin2  ---------------------------- KEY1 ---- GND
pin5  ---------------------------- KEY2 ---- GND
pin6  ---------------------------- KEY3 ---- GND

pin1(PA0/SWIM/IR) -- R2 100R ---- Gate(Q1)
Gate(Q1) -- R3 100k -------------- GND
Drain(Q1) ------------------------ IR_LED cathode
IR_LED anode -- R1 39R ----------- VDD
Source(Q1) ----------------------- GND

pin8 ----------------------------- DBG_STRAP(optional, default pull-up by firmware)
pin7 ----------------------------- TEST/RESERVED(optional)

TP_VDD -> VDD
TP_GND -> GND
TP_SWIM -> pin1
TP_IR -> Q1 gate or drain
TP_DBG -> pin8(optional)
```

## 13.2 PCB 布局建议

1. `C1/C2` 必须靠近 MCU VDD/VSS。
2. `Q1 + R1 + IR_LED + C3` 尽量靠近，减小红外脉冲环路。
3. `TP_SWIM` 放在易接探针的位置。
4. `TP_IR` 建议至少保留 1 个，方便直接量波形。
5. 按键走线尽量远离 IR 大电流回路，降低干扰。
6. 地线尽量连续，IR LED 回流不要绕 MCU 供电去耦路径。

## 14. 调试与验证方案

## 14.1 首轮上板验证顺序

1. **不上 IR LED 或不装 Q1**
   - 先验证 MCU 可通过 SWIM 稳定烧录；
   - 验证调试版固件能保留 SWIM 窗口。
2. **验证 38 kHz 载波**
   - 在 `TP_IR` 观察栅极波形；
   - 确认频率约 `38 kHz`，占空比约 `1/3`。
3. **验证 NEC 包络**
   - 观察完整 leader、bit0、bit1 的时序宽度；
   - 先测 Q1 gate，再测 IR LED 电流波形。
4. **验证按键功能**
   - KEY1/2/3 分别触发正确码值；
   - 长按时输出 repeat。
5. **验证功耗**
   - 待机电流；
   - 发射峰值电流；
   - 电池电压跌落。
6. **验证距离**
   - 室内 3~5 m 起步；
   - 若不足，再评估 `R1`/`C3`/IR LED。

## 14.2 调试判据

| 项目 | 目标 |
|---|---|
| SWIM 连接 | 调试版可稳定重连，不锁死 |
| 38 kHz 频率 | `38 kHz ± 5%` 初版可接受 |
| 占空比 | `25% ~ 40%` 初版可接受 |
| NEC bit 宽度 | `560 us / 1690 us` 级别正确 |
| 待机电流 | 首版目标 `< 5 µA`，优秀目标 `< 2 µA` |
| 峰值发射电流 | 默认 `35~45 mA` 量级 |
| 有效距离 | 室内可靠 `≥ 5 m` 作为首版合理目标 |

## 14.3 如果调试失败，优先排查顺序

1. SWIM 窗口是否被过早关闭；
2. PA0 是否在启动太早被切为 IR；
3. TIM2 频率是否真是 38 kHz；
4. TIM3 包络是否正确；
5. NMOS 是否选型不合适或焊接方向错误；
6. CR2032 在发射时是否压降过大；
7. 未用脚是否悬空，导致异常漏电或干扰。

## 15. 量产与后续优化建议

## 15.1 第一版量产注意事项

- 量产程序建议区分：
  - `DEBUG_BUILD`：保留 5 s SWIM 窗口
  - `PROD_BUILD`：缩短或移除等待，但保留恢复入口
- 板上必须至少保留 `VDD/GND/SWIM` 测试点；
- 若 pin8 实现了 `DBG_STRAP`，量产前应定义其默认状态和操作方式；
- 首版不要为了省一个 MOSFET 就切到 PA0 高灌电流直驱方案。

## 15.2 后续可优化项

第二版若要继续优化，可考虑：

1. 压缩 pin7/pin8 预留，进一步减小面积；
2. 在充分验证后再评估 `HS_EN` 直驱是否值得；
3. 根据量产样机结果调整 `R1` 与 `C3`；
4. 若接收距离要求更高，可尝试更高辐射 IR LED 或双 LED 方案；
5. 若环境对 EMC/ESD 更敏感，再加针对性保护器件。

## 16. 事实 / 工程假设 / 风险汇总

## 16.1 已确认事实

- STM8L001J3M3 适合 1.8~3.6 V 供电，能直接配合 CR2032。
- 该器件有 IRTIM，TIM2 用于载波，TIM3 用于包络。
- PA0 同时复用 SWIM 与 IR_TIM。
- 本器件无独立 NRST，调试必须基于 connect-on-the-fly。
- 目标 NEC 码值已经固定。

## 16.2 工程假设

- 单颗 CR2032 在 `35~45 mA` 红外峰值电流 + 合理去耦/储能条件下可满足首版距离要求。
- 调试版保留 SWIM 时间窗可接受。
- 第一版无需 ADC 电池测量即可完成主要功能。

## 16.3 主要风险

1. **PA0 过早切 IR，导致 SWIM 失联**
   - 规避：调试版保留 5 s 窗口；保留 DBG_STRAP。
2. **CR2032 脉冲电流不够，导致距离不足**
   - 规避：R1 从 39 Ω 起步；预留 C3；优先单颗高辐射 IR LED。
3. **未用脚或预留脚处理不当，导致待机漏电**
   - 规避：固件显式配置所有未用 GPIO 状态。
4. **误把文档中的 NRST 建议照搬到 STM8L001J3**
   - 规避：明确本器件无独立 NRST，调试策略已改为 SWIM-only。

---

## 17. 最终推荐结论

如果本项目按“**STM8L001 + CR2032 + 3键 NEC**”落地，第一版最推荐的实现路线是：

1. **硬件**：
   - `PA0` 只负责调制输出，不负责直接拉大电流；
   - 用 `NMOS + 单颗 IR LED + 39 Ω` 做默认发射级；
   - `100 nF + 4.7~10 µF` 必装，`22~47 µF` 储能焊盘建议预留；
   - 三个按键分别占用独立 GPIO，内部上拉，按键接地；
   - 预留 `VDD/GND/SWIM/IR/DBG` 测试点。

2. **软件**：
   - `TIM2` 产 38 kHz 载波；
   - `TIM3` 产 NEC 包络；
   - `IR_CR` 只开 `IR_EN`，默认不开 `HS_EN`；
   - `EXTI + Halt` 做低功耗唤醒；
   - 调试版保留 SWIM 时间窗，量产版保留恢复入口。

3. **功耗目标**：
   - 待机 `< 5 µA`，力争 `< 2 µA`；
   - 峰值 IR 电流 `35~45 mA`；
   - 首版目标室内可靠距离 `≥ 5 m`。

这套方案的核心优点是：**不冒进、容易调试、容易回退、容易验证，并且已经足够支持原理图、PCB 和固件实现直接开工。**
